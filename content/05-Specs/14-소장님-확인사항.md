# 소장님 확인 사항 (개발 착수 전)

#lifespeech #checklist

**작성일:** 2026-02-19
**목적:** 개발 시작 전 소장님께 확인받아야 할 사항 정리

---

## A. 계정·결제 (이번 주 내 필요)

### A-1. 알리고 SMS

> 문자 발송 서비스 가입 + 발신번호 등록에 **심사 3~5일** 걸립니다.

- [ ] **학원 대표번호**가 뭔가요? (예: 02-XXX-XXXX)
  - 이 번호가 문자 발신번호로 등록됩니다
  - 번호가 없으면 소장님 휴대폰 번호도 가능
- [ ] **알리고 가입**은 소장님 명의로 할까요, 학원 명의로 할까요?
  - 아이디/비밀번호를 공유받아야 합니다

### A-2. 모두싸인 (전자계약)

- [ ] **모두싸인 가입 이메일** — 어떤 이메일로 가입할까요?
  - 33talkwell@gmail.com? 아니면 별도?
- [ ] 현재 모두싸인을 **이미 사용 중**인가요?
  - 사용 중이면 기존 계정 정보 공유 필요

### A-3. Google 계정

- [ ] **33talkwell@gmail.com** 비밀번호 필요합니다
  - Google Calendar 연동(강사 1:1 스케줄)에 OAuth 설정이 필요
  - 비밀번호 직접 공유가 어려우면 제 옆에서 한 번만 로그인해주시면 됩니다
- [ ] 이 계정에 **2단계 인증** 설정되어 있나요?
  - 되어 있으면 앱 비밀번호 발급이 필요할 수 있습니다

### A-4. 앱 스토어 개발자 계정

> 앱 출시 시 필요합니다. 지금 당장은 아니지만, 심사에 시간이 걸려서 미리 만들어두면 좋습니다.

- [ ] **Google Play 개발자 계정** — 1회 $25 (약 3.5만원)
  - 소장님 개인 명의? 학원(사업자) 명의?
- [ ] **Apple Developer 계정** — 연 $99 (약 14만원)
  - 소장님 개인 명의? 학원(사업자) 명의?
  - 사업자 명의면 DUNS 번호 필요 (발급 2~4주)

### A-5. 도메인

- [ ] **lifespeech.kr** vs **lifespeech.co.kr** — 어떤 게 좋을까요?
  - 연 2~3만원
  - `admin.lifespeech.kr` (관리자 웹), `api.lifespeech.kr` (서버)로 사용

## B. 반 운영 구조 (개발 전 반드시 확정)

### B-1. 반 종류 전체 목록

> 현재 운영 중인 반을 **전부** 알아야 시스템을 정확히 만들 수 있습니다.

- [ ] **그룹반** 전체 목록을 알려주세요
  - 예시: 비기너 화요(화 19:30), 비기너 토요(토 14:00), ...
  - 각 반별: **요일, 시간, 정원(몇 명), 담당 강사**
- [ ] **1:1반**은 어떻게 운영되나요?
  - 고정 시간이 있나요, 아니면 매번 강사와 협의?
  - 1회 수업 시간이 **2시간**이 맞나요?
- [ ] **소장전담반**은 그룹반과 구조가 같나요?
  - 요일/시간 고정? 정원? 회차 수?

### B-2. 회차 수

- [ ] **그룹반 = 7회차**가 맞나요?
- [ ] **소장전담반**도 7회차인가요?
- [ ] **1:1반**은 **6회** vs **12회** 중 어떤 기준으로 나뉘나요?
  - 수강생이 선택? 가격 패키지별? 과정별?
  - 6회와 12회 이외에 다른 회차 수도 있나요?

### B-3. 수강료

> 전자계약서에 금액이 들어가므로 파악이 필요합니다.

- [ ] 과정별 **수강료**는 얼마인가요?
  - 그룹반 7회: ?원
  - 1:1 6회: ?원
  - 1:1 12회: ?원
  - 소장전담: ?원
- [ ] **예약금** 제도가 있나요? 있으면 얼마?
- [ ] **잔금** 납부 기한이 있나요? (예: 첫 수업 전까지)

## C. 출석 규칙 (비즈니스 로직)

### C-1. 상태별 규칙 재확인

> 기능명세서에 적힌 내용이 맞는지 한 번 더 확인합니다.

- [ ] **양해** — "다른 반 이동 등 사유 있음, 당일 문자 효력 없음"
  - 양해 처리되면 **회차가 소진 안 되는** 건가요?
  - 예: 3회차에 양해 → 3회차를 다시 받을 수 있나요?
- [ ] **조율** — "24시간 전 연락, 1회 보강 가능"
  - 7회차 등록 기간 중 **딱 1회만** 보강 가능한 건가요?
  - 보강 수업은 **언제** 진행하나요? (다른 반에 합류? 별도 시간?)
- [ ] **비고** — "초대, 서비스 1회"
  - 정확히 어떤 경우에 사용하나요?
  - 회차가 소진 안 되는 건가요?

### C-2. 마감 후 처리

- [ ] 7회차 마감 후 **재등록** 시, 같은 반에 연속 등록인가요?
  - 아니면 중급반 등 다른 반으로 이동 가능?
- [ ] **종료** 시 "영상 자료 이메일 송부" — 어떤 영상인가요?
  - 수업 중 촬영된 비포/애프터? 아니면 별도 교육 자료?

## D. SMS 문자 내용

### D-1. 자동 발송 문자

> 시스템이 자동으로 보내는 문자 내용을 확정해야 합니다.

- [ ] **예약 확인 문자** — 예약 등록 시 자동 발송
  - 현재 보내고 있는 문구가 있으면 공유해주세요
  - 없으면 예시: "안녕하세요 [이름]님, 라이프스피치 방문 예약이 확인되었습니다. 일시: [날짜] [시간]"
- [ ] **미방문 리마인드** — 예약했는데 안 온 분께
  - 언제 보내나요? (예약일 당일? 다음 날?)
  - 문구?
- [ ] **수업 회차 안내** — 회차 등록(출석부 생성) 시
  - 문구?
- [ ] **6~7회차 비포/애프터 안내** — 비포/애프터 영상 비교 요청
  - 문구?
- [ ] **휴무/설연휴 안내** — 일괄 발송
  - 기존에 보내던 문구?

### D-2. 수동 발송

- [ ] 관리자가 **직접 작성해서 보내는 문자**도 필요한가요?
  - 개별 발송? 반 전체? 전체 회원?
- [ ] 자주 쓰는 **템플릿**이 있으면 공유해주세요

## E. 전자계약서

### E-1. 계약서 원본

- [ ] 현재 사용 중인 **수강 계약서 원본** (워드/PDF) 공유해주세요
  - 모두싸인에 템플릿으로 등록해야 합니다
- [ ] **과정별로 계약서가 다른가요?**
  - 그룹반용 / 1:1용 / 소장전담용 별도?
  - 아니면 하나의 계약서에 과정만 체크?
- [ ] 계약서에 들어가는 **필드** 확인
  - 수강생 이름, 연락처, 과정명, 수강료, 시작일 — 이 외에 뭐가 있나요?
  - 환불 규정이 계약서에 포함되나요?

### E-2. 환불 규정

> 기능명세서: "1회 이상 수업 시 환불 불가. 불가피 시 운영비 10% 공제"

- [ ] 이 규정이 **현재 기준과 동일**한가요?
- [ ] **양도** — "3개월 내 사용, 대체 처리" 라고 되어있는데
  - 양도 대상은 아무나? 가족만?
  - 양도 수수료가 있나요?

## F. 기존 데이터 이전

### F-1. 회원 데이터

- [ ] 현재 회원 관리를 **어떤 형태**로 하고 있나요?
  - 엑셀? 구글 시트? 노션? 종이?
- [ ] 해당 파일을 **공유**해주세요 (개발용 이전 스크립트 작성 필요)
  - 최소 필요 정보: 이름, 전화번호, 등록 과정, 등록일
- [ ] 현재 기준 **총 회원 수**가 대략 몇 명인가요?
- [ ] 기존 회원 중 **현재 수강 중인 분**은 몇 명인가요?

### F-2. 강사 데이터

- [ ] **강사 전체 명단**을 주세요 (이름, 이메일, 담당 반)
  - 시스템에 강사 계정을 생성해야 합니다
- [ ] 강사마다 **Google Calendar**를 쓰고 있나요?
  - 쓰고 있으면 각 강사의 Gmail 주소 필요 (1:1 스케줄 연동)
  - 안 쓰고 있으면 Google Calendar 연동은 나중으로 미룰 수 있음

### F-3. 관리자 계정

- [ ] **관리자 계정**은 몇 개 만들까요?
  - 이름, 이메일 목록 (예: 실장님 1명, 연구원 1명 등)

## G. 개인정보·보안

### G-1. 유명인 보호

> 요구사항: "회원 정보 비공개 (유명인·국회의원 등)"

- [ ] **구체적으로 어떤 수준의 보호**가 필요한가요?
  - 이름 마스킹? (예: 홍*동)
  - 특정 회원을 "비공개" 플래그로 설정 → 관리자만 열람?
  - 강사도 해당 회원 이름을 못 보나요?

### G-2. 개인정보 보관

- [ ] **개인정보 처리방침**이 이미 있나요?
  - 앱 스토어 제출 시 필수입니다
  - 없으면 제가 기본 양식을 만들어드립니다
- [ ] 회원 탈퇴 시 데이터 **보관 기간**은?
  - 즉시 삭제? 1년? 3년?

## H. 기능 우선순위 최종 확인

### H-1. 출시 때 꼭 필요한 것

> 아래 기능이 **1차 출시에 반드시** 들어가야 하는지 확인합니다.

- [ ] **전자계약 (모두싸인)** — 출시 때 꼭 필요? 아니면 처음에는 수동으로 해도 됨?
- [ ] **Google Calendar 1:1 매칭** — 출시 때 꼭 필요? 아니면 수동 스케줄로 시작해도 됨?
- [ ] **강사 체크리스트/평가** — 출시 때 꼭 필요? 아니면 일단 출석 관리만으로 시작해도 됨?
- [ ] **숙제·영상 업로드** — 출시 때 꼭 필요? 파일 용량 많으면 비용 이슈 있음

### H-2. 반 밴드 (커뮤니티)

> 플로우차트에 네이버 밴드 스타일의 "반 밴드" 기능이 있는데, 다른 문서에는 없습니다.

- [ ] 이 기능은 **출시 후에 추가해도 괜찮나요?** (P2로 분류했습니다)
  - 현재는 네이버 밴드 등을 대신 사용하고 있나요?
