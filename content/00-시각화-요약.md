# LifeSpeech 시각화 요약

#visualization #overview #lifespeech

**목적:** 한눈에 보는 그래프·다이어그램 모음. 브리핑 시 참고.

---

## 1. 프로젝트 한눈에

```mermaid
flowchart TB
    subgraph 입력["입력"]
        A1[네이버 예약]
        A2[방문 체크]
    end
    
    subgraph 처리["처리"]
        B1[등록/미등록/대기]
        B2[반 배정]
        B3[전자계약]
        B4[출석부]
        B5[회차별 문자]
    end
    
    subgraph 출력["출력"]
        C1[재등록/종료]
        C2[회원 관리]
        C3[강사 체크리스트]
    end
    
    A1 --> A2 --> B1 --> B2 --> B3 --> B4 --> B5 --> C1
    B4 --> C2
    B4 --> C3
```

---

## 2. 역할·메뉴 구조

```mermaid
flowchart TB
    subgraph 앱["1개 앱"]
        ADMIN[관리자]
        INSTR[강사]
        MEMBER[회원]
    end
    
    ADMIN --> A1[예약·등록·출석]
    ADMIN --> A2[회원·강사 관리]
    ADMIN --> A3[스케줄·문자]
    
    INSTR --> I1[내 일정]
    INSTR --> I2[체크리스트]
    INSTR --> I3[파일 교환]
    
    MEMBER --> M1[내 수업]
    MEMBER --> M2[숙제·영상 업로드]
```

## 3. 핵심 플로우 (예약→출석→마감)

```mermaid
flowchart LR
    A[네이버 예약] --> B[방문 체크]
    B --> C{등록?}
    C -->|그룹| D[7회차 자동]
    C -->|1:1| E[캘린더 매칭]
    C -->|미등록| F[마케팅]
    D --> G[출석부]
    E --> G
    G --> H[회차별 문자]
    H --> I{7회 완료?}
    I -->|Yes| J[재등록/종료]
```

## 5. 기술 스택

```mermaid
flowchart TB
    subgraph Front["프론트엔드"]
        NEXT[Next.js]
        RN[React Native]
    end
    
    subgraph Back["백엔드"]
        API[Fastify API]
    end
    
    subgraph Data["데이터"]
        DB[(Supabase PostgreSQL)]
        STORAGE[(Supabase Storage)]
    end
    
    subgraph Infra["인프라"]
        VERCEL[Vercel]
        RAILWAY[Railway]
    end
    
    NEXT --> API
    RN --> API
    API --> DB
    API --> STORAGE
    NEXT --> VERCEL
    API --> RAILWAY
```
